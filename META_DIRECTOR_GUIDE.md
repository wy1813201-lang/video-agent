# Meta Director 使用指南

## 简介

Meta Director 是一个简化版的"老板 Agent"，负责审核内容质量、管理实验版本、记录决策数据。

## 功能特性

### 1. 质量审核系统

Meta Director 会在关键节点自动审核内容质量：

- **剧本审核**：检测 Hook 吸引力、剧情结构、情绪节奏
- **分镜审核**：检测镜头逻辑、镜头数量、角色一致性
- **视频审核**：检测视觉质量、时长合理性

### 2. 决策系统

根据评分自动做出决策：

- **APPROVE（通过）**：分数 ≥ 7.0，继续下一步
- **REJECT（退回）**：分数 < 5.0，需要重新生成
- **EXPERIMENT（实验）**：分数在 6.0-8.0 之间，生成多个实验版本
- **MANUAL（人工）**：需要人工介入

### 3. 实验版本系统

当内容质量在及格线附近时，自动生成多个实验版本：

**剧本实验参数**：
- 冲突时间（immediate/early）
- 反转次数（2-3次）
- 情绪强度（high/extreme）
- Hook风格（shock/mystery）

**分镜实验参数**：
- 镜头数量（+20%/-10%）
- 特写比例（30%-50%）
- 运镜风格（dynamic/stable）
- 剪辑速度（fast/medium）

**视频实验参数**：
- 调色风格（warm/cinematic）
- 转场风格（dissolve/fade）
- 音乐强度（high/medium）
- 字幕风格（bold/elegant）

### 4. 数据记录系统

记录每次生产的完整数据：

```json
{
```

## 使用方法

### 1. 启用 Meta Director

编辑 `config.yaml`：

```yaml
meta_director:
  enabled: true               # 启用 Meta Director
  min_score: 7.0              # 最低通过分数
  enable_experiments: true    # 启用实验版本
  experiment_count: 2         # 生成2个实验版本
  records_dir: "data/production_records"
```

### 2. 运行生成流程

```bash
python main.py generate --topic "重生千金复仇记" --style 情感 --episodes 3
```

### 3. 查看审核结果

运行时会显示审核信息：

```
📝 第 1 集
   ✓ 剧本生成完成 (2847 字)

   🎯 [Meta Director] 剧本审核
      决策: approve
      评分: 8.5/10
      理由: 质量良好，总分8.5/10

   ✓ 两步分镜生成: 12 个镜头 → output/storyboards/storyboard_flow_ep01.json

   🎯 [Meta Director] 分镜审核
      决策: approve
      评分: 8.2/10
      理由: 质量良好，总分8.2/10
```

### 4. 查看生产记录

生产记录保存在 `data/production_records/` 目录：

```bash
ls data/production_records/
# 重生千金复仇记_ep01_20260304_143022.json
```

### 5. 手动录入平台数据（可选）

生成完成后，可以手动录入平台表现数据：

```python
from src.meta_director import MetaDirector

director = MetaDirector()
# 加载记录
# director.current_record = ...

# 更新平台数据
director.update_platform_data({
    "views": 125000,
    "completion_rate": 0.78,
    "likes": 8500,
    "comments": 320,
    "shares": 156
})
```

## 评分标准

### 剧本评分（0-10分）

**Hook 吸引力（0-10）**：
- 基础分：5.0
- 开头100字内有强冲突关键词：+1.0/个
- 开头有对话：+1.0

**剧情结构（0-10）**：
- 基础分：5.0
- 场景数量3-5个：+2.0
- 冲突关键词：+0.5/个（最多2.0）
- 反转关键词：+0.3/个（最多1.0）

**情绪节奏（0-10）**：
- 基础分：5.0
- 每种情绪类型：+1.0（愤怒/悲伤/惊讶/喜悦）

### 分镜评分（0-10分）

**镜头逻辑（0-10）**：
- 基础分：5.0
- 有建立镜头：+2.0
- 镜头类型多样性：+0.5/种（最多3.0）

**镜头数量（0-10）**：
- 8-15个：10.0
- 5-8个或15-20个：7.0
- 其他：5.0

**角色一致性（0-10）**：
- 基础分：5.0
- 有面部特征描述：+1.0/角色
- 有服装风格描述：+1.0/角色

### 视频评分（0-10分）

**视觉质量（0-10）**：
- 基于技术质量评分（文件大小、时长、分辨率）

**时长合理性（0-10）**：
- 50-70秒：10.0
- 40-50秒或70-80秒：7.0
- 其他：5.0

## 实验版本工作流

当触发实验模式时：

1. **生成基础版本**
2. **Meta Director 评分**：6.0-8.0 之间
3. **触发实验模式**：生成2个变体版本
4. **人工选择**：从多个版本中选择最佳
5. **记录选择原因**：积累数据

示例：

```
🧪 [ExperimentEngine] 开始实验: script
   基础内容: str
   实验版本数: 2

   [版本 1] script_v1
   参数: {'name': '快节奏版', 'conflict_timing': 'immediate', ...}
   ✓ 生成成功

   [版本 2] script_v2
   参数: {'name': '标准版', 'conflict_timing': 'early', ...}
   ✓ 生成成功

✓ 实验完成: 成功生成 2/2 个版本
```

## 数据积累

Meta Director 会记录：

- 每次审核的评分和决策
- 实验版本的参数和结果
- 人工选择的版本和原因
- 平台表现数据（手动录入）

这些数据为未来的 AI 学习和优化提供基础。

## 注意事项

1. **简化版限制**：
   - 使用规则based审核，不依赖 LLM
   - 实验版本生成是占位实现，需要进一步完善
   - 平台数据需要手动录入

2. **成本考虑**：
   - 启用实验模式会增加 API 调用成本
   - 建议先在小规模测试中使用

3. **人工介入**：
   - 当前版本仍需要人工选择最佳版本
   - 未来可以训练 AI 模型自动选择
